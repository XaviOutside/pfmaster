# Architecture Decision Records (ADR)

This directory contains Architecture Decision Records — lightweight documents that capture significant architectural decisions, the context that led to them, and their consequences.

## Format

Each ADR follows this structure:

```
# ADR-NNNN: Title

**Status**: proposed | accepted | deprecated | superseded

**Date**: YYYY-MM-DD

## Context
What is the issue that we're seeing that is motivating this decision?

## Decision
What is the change that we're proposing and/or doing?

## Consequences
What becomes easier or more difficult to do because of this change? Includes tradeoffs, risks, and follow-up actions.
```
